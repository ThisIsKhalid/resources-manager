AI Engineer — 4-Day Plan

Eid holiday study plan for fullstack developers  ·  3–4 hrs/day

1

Python for JS devs + AI landscape

3–4 hours  ·  No prior Python needed

Python Jupyter AI overview Morning
Block 1  ·  1 hr
Python crash course
Syntax, list comprehensions, dicts, f-strings, pip. Think of it as JS but with significant whitespace.
Block 2  ·  45 min
Jupyter + virtual env setup
Install Jupyter, create a venv, run your first notebook cell. This is your daily workbench.
Block 3  ·  1 hr
How LLMs actually work
Tokens, context windows, temperature, system prompts. Know these before touching any API.
Block 4  ·  30 min
AI engineer landscape
Where you fit: app layer vs model layer. Tools: LangChain, LlamaIndex, OpenAI SDK, Claude API.

MINI PROJECT

End of day goal
Write a Python script that reads a .txt file and prints word count
  • Use open(), read(), split() — no AI yet
  • Run it in Jupyter notebook
  • Gets you comfortable before tomorrow's API calls
2

LLM APIs + prompt engineering

3–4 hours  ·  Your REST API skills kick in here

OpenAI SDK Claude API Prompt engineering Streaming
Block 1  ·  1 hr
Call your first LLM API
Set up OpenAI or Anthropic SDK. Send a message, read the response. Compare to a raw REST call — you already know this shape.
Block 2  ·  1 hr
Prompt engineering patterns
System prompts, few-shot examples, chain-of-thought, output formatting with JSON mode. These determine 80% of your app quality.
Block 3  ·  45 min
Streaming responses
Stream tokens to the frontend in real time. Critical for any chat UI — users hate waiting for a full response.
Block 4  ·  30 min
Tool use / function calling
Tell the model to call your functions. This is how AI apps take real actions — search, write files, call APIs.

MINI PROJECT

End of day goal
Build a CLI chatbot with memory of the last 5 messages
  • Use the messages array to maintain conversation history

3

RAG — give your app knowledge

3–4 hours  ·  The most in-demand AI skill right now

Embeddings Vector DB RAG pipeline LangChain
Block 1  ·  1 hr
Embeddings explained
Turn text into vectors. Understand semantic similarity — why "cat" and "kitten" are close but "car" is far.
Block 2  ·  1 hr
Vector databases
Store and search vectors with ChromaDB (local, zero config). Learn similarity search — your AI's long-term memory.
Block 3  ·  1 hr
Build a RAG pipeline
Load docs → chunk → embed → store → retrieve → generate. The core pattern behind every "chat with your data" app.
Block 4  ·  30 min
LangChain basics
Chains, document loaders, text splitters. Use it to simplify your RAG code — but understand what it abstracts.

MINI PROJECT

End of day goal
Chat with a PDF — Q&A over your own document
  • Load a PDF, chunk it into 500-token pieces
  • Embed chunks and store in ChromaDB
  • On each question, retrieve top 3 chunks and pass to LLM
4

Ship a full AI app

3–4 hours  ·  Bring your fullstack skills together

FastAPI Next.js / React AI agents Deploy
Block 1  ·  1 hr
FastAPI backend with LLM
Wrap your Day 2 chatbot in a FastAPI endpoint. Stream tokens via SSE — your Next.js frontend already knows how to consume this.
Block 2  ·  1 hr
Connect your frontend
Build a simple chat UI in React/Next.js. Call your FastAPI. Display streamed tokens. You know this — it's just another API.
Block 3  ·  45 min
AI agents intro
Give the LLM tools: web search, calculator, file writer. Let it decide which tool to call. This is the next frontier after RAG.
Block 4  ·  30 min
Deploy + next steps
Push to Vercel (frontend) + Railway/Render (backend). Set your API keys as env vars. Share the link.

FINAL PROJECT OPTIONS

Pick one — build and deploy it today
Option A — AI customer support bot for a fake product
  • RAG over a product FAQ (Day 3 pipeline)
  • Streaming chat UI (React + FastAPI)
  • System prompt that keeps it on-topic
Or go simpler
Option B — AI writing assistant with your own tone
  • System prompt trained on your writing style
  • Text input → improve / expand / shorten buttons
  • No RAG needed — pure prompt engineering
